## Summary

`v0.3.3` is the federation-hardening, public-eval-analytics, and full real-network-green release for `easy-agent`.
It carries forward the `0.3.2` executor base and closes the next reinforcement loop around A2A negotiation, MCP approval handling, OpenAI-compatible fallback behavior, local GitHub automation hooks, and host-level real-network verification.

## Highlights

- Expanded A2A-style federation compatibility with richer negotiation metadata, security hints, callback verification, and remote readiness checks.
- Strengthened MCP human-loop handling with structured form approvals, risk-aware deferred escalation, and safer public-eval fallback behavior.
- Added optional local GitHub automation helpers for issue listing, repair-package preparation, local commit creation, and release publishing.
- Added stage-aware public-eval analytics, failure buckets, and a per-provider schema compatibility matrix.
- Strengthened regression coverage around duplicate-call suppression, history grounding, protocol/schema normalization, and callback header handling.
- Promoted the real-network matrix from partially skipped coverage to a fully green host snapshot with preflighted live-model, container, and microVM paths.
- Hardened Windows-side HTTP client cleanup so successful live-model runs are not downgraded by transport teardown noise.
