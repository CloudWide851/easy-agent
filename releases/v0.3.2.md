# easy-agent v0.3.2 Release Notes

Release date: 2026-03-27
Tag: `v0.3.2`
Release commit: `a5310d1` (`feat(runtime):增加执行器后端与真实网络评测`)

## Summary

`v0.3.2` is the executor-backend and real-network-evaluation release for `easy-agent`.
It turns executor isolation into a first-class runtime capability, adds a published real-network matrix, and hardens OpenAI-compatible schema handling so long-running and MCP-backed paths stay stable under provider-specific tool-calling constraints.

## Included Commit

- `a5310d1` `feat(runtime):增加执行器后端与真实网络评测`

## Highlights

- Added named executor backends for `process`, `container`, and `microvm` execution paths.
- Extended workbench reuse so long-lived command skills and stdio MCP sessions can bind to named executors and persist runtime state.
- Added `src/agent_runtime/real_network_eval.py` and the corresponding real integration coverage to publish a host-facing real-network matrix.
- Hardened OpenAI-compatible tool-schema sanitization for `anyOf`, `oneOf`, list-typed `type`, and format-heavy schemas.
- Adjusted filesystem MCP roots negotiation so stdio filesystem servers keep using configured allowed directories safely.
- Refreshed the bilingual README pair and `CHANGELOG.md` for the `0.3.2` release snapshot.
